### FinChina-SA

- 来源：https://github.com/YerayL/FinChina-SA
- 许可证：Apache-2.0
- 本地目录：`raw/FinChina-SA-main/`
- 训练集：9,933 条
- 测试集：1,103 条
- 合计：11,036 条
- 字段：`newscode`、`title`、`text`、`institution`

适合训练或评估中文金融事件的细粒度情绪与机构影响，不是现成的问答数据。转换成 SFT 数据前需要明确标签语义，并人工抽查新闻与标签的一致性。

### CFEED

- 来源：https://github.com/brickee/CFEED
- 许可证：MIT
- 本地目录：`raw/CFEED-data/`
- 合计：7,144 条
- 字段：`ID`、`Title`、`Doc`、`Event`

任务分布：

| 文件 | 样本数 | 事件 |
| --- | ---: | --- |
| `FREEZE_Data.json` | 1,039 | 股权冻结 |
| `PLEDGE_Data.json` | 4,202 | 股权质押 |
| `OW&UW_Data.json` | 1,903 | 股东增持与减持 |

CFEED 使用远程监督自动对齐公告与事件模板，适合训练事件抽取，但存在标签噪声，不能未经抽查就训练成投资结论。

## 质量原则

- 原始金融文本不等于正确投资答案。
- 情绪标签不能直接转换成涨跌标签。
- 行业政策必须记录发布日期、实施日期、政策方向和原始来源。
- 所有涉及未来收益的数据必须按时间切分，避免未来信息泄漏。
- 训练集、验证集和测试集应按来源或时间去重，而不是简单随机抽样。
- 鼓手最终需要自己积累人工审核的高质量问答和纠错集。

## 建议转换任务

1. `DISC consulting/retrieval`：转换成鼓手问答格式并人工修订。
2. `FinChina-SA`：转换成“事件影响方向 + 理由 + 不确定性”的分类任务。
3. `CFEED`：转换成“公告文本 -> 事件、主体、数值、日期、风险”的结构化抽取任务。
4. 将鼓手真实使用中的好问题和人工修正版加入自有数据集。

第一次训练不应追求数量。建议先制作 1,000-3,000 条经过人工核验的鼓手数据，并保留至少 200 条完全隔离的评测题。
